# Issue Management Guidelines

Issues from this repo sync automatically to our central Notion tracker. Follow these guidelines so issues appear correctly.

---

## Labels → Status

| GitHub Label | Notion Status |
|--------------|---------------|
| *(no label)* | Backlog |
| `in-progress` | In Progress |
| `blocked` | Blocked |
| `review` or `under-review` | Under Review |
| *(issue closed)* | Done |

## Labels → Priority

| GitHub Label | Notion Priority |
|--------------|-----------------|
| `high-priority` or `high` or `urgent` | High |
| `medium` or `medium-priority` | Medium |
| `low` or `low-priority` | Low |
| *(no priority label)* | Medium |

---

## Milestones → Due Dates

Set a **milestone with a due date** on your issue and it will sync to Notion's Due Date field.

---

## Assignees

Assign yourself or others to the issue - it syncs to Notion's Assignee field.

---

## Quick Reference

When creating an issue:
1. **Title**: Clear, actionable description
2. **Labels**: Add priority (`high`, `medium`, `low`) and status (`in-progress`, `blocked`, `review`) as needed
3. **Milestone**: Add if there's a deadline
4. **Assignee**: Assign to the responsible person

---

## Sync Frequency

- Issues sync to Notion every **30 minutes** automatically
- Changes in Notion sync back to GitHub every **15 minutes**
